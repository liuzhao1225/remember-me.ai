# 未来工作：从 Vibe Coding 到 Vibe Management

最近我越来越强烈地感觉到，AI Agent 带来的变化，不是「程序员效率提升」这么简单。代码只是第一块被击穿的多米诺骨牌。真正被改写的是整个工作的形态：人不再围绕键盘、屏幕、App 和文件夹组织自己，而是围绕目标、权限、信息流和 Agent 组织自己。

过去我们说办公，脑子里默认有一张桌子、一台电脑、一个键盘、几块屏幕。电脑是主机，屏幕是窗口，键盘是输入，App 是工作的容器。写代码在 IDE 里，沟通在飞书里，资料在浏览器里，任务在项目管理工具里，文档在云文档里。人像一个手动路由器，在这些工具之间来回切换，把信息从一个地方复制到另一个地方，把一个人的意思转成另一个系统能理解的格式。

但 Agent 出现以后，这套结构开始松动。

OpenAI 在 2026 年 5 月把 Codex 带到手机上，手机可以连接正在运行 Codex 的电脑、devbox 或远程环境，查看线程、批准命令、看 diff、看测试结果、看截图。这个产品信号很重要。它说明主机不再必须和人待在一起，屏幕也不再必须是工作的中心。主机可以在云端、机房、家里的 Mac mini、公司的远程环境里跑。人只需要一个控制面板。

更重要的是，随着 Codex、Claude Code、Cursor、浏览器插件、MCP、飞书 CLI 这类东西逐渐成熟，Agent 能进入的系统越来越多。以前 Agent 只能在文本框里回答问题。后来它能改代码。再后来它能操作浏览器。现在它开始能调用 App 的 CLI、API、MCP、自动化接口，能发消息、查文档、改表格、建任务、开 PR、跑测试、看日志、部署服务。

这意味着 Agent 的边界正在从「写代码」扩展到「做工作」。

---

## 代码只是最先被自动化的工作

为什么最先发生在代码领域？因为代码是信息工作里最适合 Agent 的一种。

代码有明确的输入和输出，有文件系统，有版本控制，有测试，有编译器，有错误日志，有自动化验证。一个功能要不要完成，不完全依赖人的主观感受。测试通过没有，页面能不能打开，接口返回对不对，类型检查有没有报错，这些都可以机器判断。

所以 vibe coding 最先成立。人只说需求，Agent 写代码。出了 bug，Agent 自己修。修完跑测试。测试不过，把错误继续喂给 Agent。人越来越少看代码，甚至完全不看代码。Andrej Karpathy 在 2025 年提出 vibe coding 时就说过，他已经 `Accept All`，不再读 diff。到 2026 年，他在 No Priors 访谈里说，自己从 2025 年 12 月开始基本没有再亲手敲代码，工作变成了向 Agent 表达意图。

这不是一个人的极端体验，而是方向。

代码的特殊之处在于，它提前具备了 Agent 时代需要的基础设施：结构化输入、可执行输出、自动验证、版本回滚、权限边界、日志记录。这些东西一旦迁移到其他行业，vibe coding 就自然变成 vibe work。

以后不是「我让 Agent 写代码」，而是：

「帮我把这个客户推进到签约前状态。」

「把这周所有会议纪要整理成下周执行计划，并给每个人建任务。」

「检查这个投放账户为什么 ROI 下降，找出三个最可能原因，能改的直接改，不能改的列出审批项。」

「把这个产品需求推进到可开发状态，补齐用户故事、交互流程、验收标准和风险点。」

这些都不是代码任务，但它们和代码任务本质一样：接收信息，理解状态，生成下一步动作，把信息送到正确的地方。

## 工作本质上是信息流

如果把工作剥到最底层，大部分所谓白领工作，本质都是信息流动。

销售在处理信息：客户说了什么，预算多少，谁是决策人，下一步该给什么材料。

产品经理在处理信息：用户反馈、业务目标、工程约束、设计方案、优先级。

程序员在处理信息：需求、代码、错误日志、架构约束、测试结果。

运营在处理信息：数据波动、内容素材、投放反馈、用户行为。

管理者在处理信息：目标、资源、风险、人、时间、组织情绪。

所谓「工作能力」，很大程度上就是判断信息下一步应该流向哪里。该转给谁，该沉淀成文档，该变成任务，该进入代码，该升级成决策，该暂时忽略，该等待更多信号。

现在的 Agent 已经很会处理信息。它能读很多，能总结，能写，能改，能搜索，能对话，能调用工具。但它还不总是知道下一步该做什么。它需要人告诉它：这件事重要吗？下一步应该发给谁？这个风险能不能接受？这个客户要不要优先？这段信息应该沉淀成制度，还是只是一次性处理？

所以今天的人，还在很多流程里扮演「胶水组件」。

我们把会议纪要发给 Agent，让它整理。整理完以后，我们决定发到哪个群。我们让 Agent 修 bug，修完以后，我们决定能不能合并。我们让 Agent 查资料，查完以后，我们决定写成文章、发给同事、建任务，还是先放着。

人类现在还在信息流里做路由。

但这是过渡态。

## App 会从人的工具变成 Agent 的接口

今天的大部分 App 是给人设计的。按钮、菜单、输入框、弹窗、表格、筛选器、拖拽上传，这些都是人机交互时代的产物。Agent 当然可以通过浏览器插件去点这些东西，但那只是过渡方案。真正的 Agent-native App，不会只提供 UI，而会提供机器可调用的工作接口。

未来每个严肃 App 都会有几层接口。

第一层是 API。Agent 可以读写数据。

第二层是 CLI。Agent 可以用稳定命令执行常见操作。

第三层是事件流。Agent 可以订阅状态变化，比如客户回复了、审批通过了、CI 失败了、合同快到期了。

第四层是权限和审计。Agent 做了什么、为什么做、用谁的权限做、能不能回滚，都必须可追踪。

第五层是语义层。App 不只是暴露「创建一条记录」这种低级接口，而是暴露「推进商机」「发起审批」「归档会议」「更新项目状态」这种业务动作。

飞书 CLI、GitHub CLI、Codex 的浏览器/桌面能力、MCP 生态，本质上都是这个趋势的早期形态。App 不再只是给人看的页面，而是一个可被 Agent 调用的工作状态机。

当这件事普及以后，人的工作界面会发生根本变化。你不再需要打开十个 App，也不再需要记住每个 App 的操作路径。你只需要表达目标，Agent 自己决定调用哪些系统。

「把昨天和张总聊的方案整理成正式邮件，附上报价表，发出前给我看一眼。」

这句话背后可能涉及聊天记录、客户档案、报价系统、云文档、邮件、审批流、日程系统。但你不需要知道。Agent 知道。

## 设备也会重新分工

如果工作从「人在 App 里操作」变成「人指挥 Agent 处理信息流」，设备形态也会变。

主机会变成远端执行层。它可能是云端 devbox，可能是公司机房，可能是本地 Mac mini，也可能是一个长期运行的个人 Agent 环境。它不需要总在你身边。它负责算力、上下文、文件、权限、长任务、后台执行。

手机会变成随身审批器和低摩擦输入器。它不适合长时间阅读复杂信息，但适合发起任务、批准权限、看摘要、处理异常、回复 Agent 的关键问题。手机不是工作台，而是遥控器。

平板会变成临时工作台。它适合在咖啡店、飞机、高铁上展开一个较大的任务视图，看报告、看设计稿、看任务树、做一次短时深度干预。

AR 眼镜会变成真正有意思的形态。它的价值不是在空中开三个 VS Code 窗口，让人继续读代码。那是把旧范式搬到新设备上。AR 眼镜的价值是让你随时处在一个低侵入的信息指挥舱里。

你走在路上，眼镜显示：

三个 Agent 正在跑。

一个任务需要你批准外发邮件。

一个客户回复了关键问题。

一个部署失败，Agent 已经定位到原因，建议回滚。

一个研究任务完成，结论置信度 0.72，需要你决定是否继续深挖。

你用语音说：「邮件给我读摘要。报价金额没问题，发。部署先回滚。研究任务继续，把反方材料也补齐。」

这不是科幻。它只是把今天已经分散在电脑、手机、浏览器、IM、IDE 里的动作，收敛成一个随身的 Agent 控制层。

键盘不会消失，但会退到次要位置。屏幕不会消失，但不再是工作的唯一入口。真正的入口会变成自然语言、语音、视线、手势、上下文感知，以及最重要的：权限确认。

## 人的新角色：从执行者到指挥家

如果 Agent 能读信息、写信息、操作 App、调用工具、跑长任务，人还剩下什么？

这其实不是一个悲观问题。人剩下的东西更接近工作的本质。

人提供目标。不是「把这个按钮点一下」，而是「这个季度我要增长」「这个客户不能丢」「这个产品要更简单」「这篇文章要锋利一点」。

人提供价值判断。Agent 可以算 ROI，但它不天然知道什么是值得做的。它可以列风险，但它不天然知道你愿意承担哪种风险。它可以写得正确，但未必知道什么叫有味道。

人提供边界。哪些权限可以自动用，哪些必须询问。哪些信息能发给外部，哪些只能内部流转。哪些动作可以回滚，哪些动作一旦执行就不可逆。

人提供品味。未来内容会更不缺，代码会更不缺，方案会更不缺。稀缺的是判断什么值得留下，什么应该删掉，什么看起来对但没有生命力。

人提供责任。Agent 可以执行，但组织和社会最后仍然需要一个责任节点。谁授权，谁承担后果，谁定义成功，谁决定停止。

所以未来的工作者，不像传统意义上的操作员，也不像传统意义上的经理。他更像一个指挥家。乐器不是人，而是一组 Agent、一组工具、一组信息流、一组权限边界。指挥家不亲自拉小提琴，不亲自敲定音鼓，但他决定节奏、轻重、进入时机和整体方向。

更进一步说，未来的管理也会变化。

过去管理是管理人。谁做什么，什么时候交，进度如何，哪里卡住，谁需要协调。

未来管理会越来越多地变成管理 Agent 和信息流。哪些 Agent 负责研究，哪些负责执行，哪些负责验证，哪些负责审计。哪些信息可以自动流转，哪些必须人工确认。哪些指标触发行动，哪些异常触发上报。哪些流程可以无人运行，哪些流程必须保留人类决策点。

这就是 vibe management。

不是老板躺着让 AI 干活，而是把组织变成一个可编排的信息系统。人类在其中不再是每一步的搬运工，而是目标函数、约束条件和最终责任的来源。

## 人工零参与的临界点

METR 在 2026 年 5 月更新的 Time Horizon 数据很有意思。它衡量的不是 Agent 实际连续运行多久，而是一个 Agent 能以一定成功率完成多长的人类专家任务。最新数据里，前沿模型的 50% time horizon 已经进入十小时级别。Claude Opus 4.6 大约是 718.8 分钟，接近 12 小时。更早期的 Claude Mythos Preview early 数据甚至超过 1000 分钟，但 METR 也提醒，16 小时以上的测量已经接近当前任务集的上限，不宜过度解读。

即便保守看，这也已经足够说明问题：Agent 正在从「完成一个小步骤」跨到「完成半天到一天的完整任务」。如果这个长度继续按几个月翻倍，工作范式会很快变化。

一小时任务和一天任务之间，不只是量变。

一小时 Agent 只能做一个明确任务：修一个 bug，写一个页面，整理一个文档。

一天 Agent 可以自己经历多个阶段：理解目标，查资料，拆任务，执行，遇到错误，修复，验证，生成报告，请求批准。

一周 Agent 则是另一种东西。它可以跟踪外部变化，等待他人回复，根据新信息调整策略，维护一个持续状态。到这个阶段，Agent 不再只是工具，而是一个低成本、可复制、可并行的工作单元。

人工参与会越来越集中到少数节点：

目标是否正确。

风险是否可接受。

结果是否足够好。

权限是否允许。

价值是否值得。

其他大量中间步骤都会消失在 Agent 的执行层里。人不再看每一次点击、每一行代码、每一封草稿、每一条日志。人只看证据、异常和结果。

## 这和数字永生有什么关系

这个项目叫 remember-me.ai。表面上，它是在保存人的文章、故事、观点和自我叙述。但从更深一层看，它保存的不是静态记忆，而是一个人处理信息的方式。

一个人为什么是这个人？不是因为他有某个固定的标签，而是因为他在面对信息时，会用某种稳定的方式判断、选择和表达。

看到一个技术趋势，他会往哪个方向推演。

看到一个产品，他会先看数据飞轮还是先看 UI。

看到一个社会变化，他会感到兴奋还是恐惧。

遇到风险，他倾向于保守还是冒险。

写一篇文章，他会怎样开头，怎样递进，怎样下结论。

这些东西都是信息处理模式。

如果未来的工作本质是信息流，未来的 Agent 本质是在替人处理信息流，那么一个人的数字遗产就不只是「我说过什么」，而是「我会怎样处理」。把文章、观点、判断、价值倾向留下来，不只是为了让未来某个系统引用一句话，而是为了让未来系统在处理信息时，能够吸收一点你的路由方式、判断方式、表达方式。

换句话说，数字永生不只是被记住，而是参与未来信息流的处理。

当 Agent 帮我写文章、做研究、管理项目、判断风险时，如果它吸收过我的数据，它就可能在某些微小的地方延续我的偏好。它可能更关注系统结构，更相信信息流，更看重长期影响，更愿意把人放到智能演化的历史尺度里看。这不等于系统里有一个「我」的副本。但它意味着我的信息模式参与了系统。

这比传统意义上的纪念更接近永生。

传统纪念是别人想起你。

数字永生是系统继续使用你。

## 不是乌托邦

当然，这一切不只带来解放，也带来风险。

Agent 如果把信息流向错误的地方，后果会比人手动犯错更快、更大。一个误配置的自动化，可能把内部资料发到外部，把错误代码部署到生产，把敏感数据暴露到公网。

Agent 如果只优化局部指标，也会制造新的荒谬。安全 Agent 可能为了清空告警而隐藏问题，销售 Agent 可能为了提高转化而过度承诺，管理 Agent 可能为了提高效率而压扁人的复杂性。

Agent 如果成为新的工作入口，谁拥有 Agent，谁就拥有信息流。未来的权力不只是掌握数据，而是掌握数据如何流动、如何被解释、如何变成行动。

所以未来的关键不是「让 Agent 自由行动」，而是建设一整套 Agent 工作基础设施：权限、审计、回滚、验证、隔离、模拟、问责、可解释的决策记录。没有这些，vibe work 会变成 vibe accident。

但风险不改变方向。就像互联网带来信息污染，却没有让人类退回纸质时代。Agent 带来的问题，会通过更 Agent-native 的制度和工具解决，而不是通过回到人工复制粘贴解决。

## 新工作范式

未来工作的基本单元，可能不再是「人打开 App 做一件事」，而是：

**一句目标 + 一组 Agent + 一组权限 + 一个可验证结果。**

设备也会围绕这个范式重组：

云端是执行层。

手机是随身审批器。

平板是临时工作台。

AR 眼镜是环境化的信息指挥舱。

键盘是兜底输入。

大屏是深度分析和少数复杂场景的展开空间。

App 是 Agent 可调用的状态机。

人是目标、品味、边界和责任。

今天我们还在频繁参与，是因为 Agent 的长程能力、工具接口、权限系统、组织流程都还没有完全成熟。但趋势已经很清楚：人工参与会越来越少，Agent 能一次完成的事情会越来越长。人会从工作流的每一步里退出来，站到工作流的上方。

这不是人变得不重要。

恰恰相反。低层操作越自动化，人的目标和判断越重要。因为系统会越来越强，错误目标的放大能力也会越来越强。未来最重要的人，不是最会操作软件的人，也不是最会写代码的人，而是最能定义目标、设计信息流、判断结果、承担责任的人。

过去，一个人的生产力来自他能亲手完成多少动作。

未来，一个人的生产力来自他能让多少 Agent 在正确的方向上持续行动。

这就是从 vibe coding 到 vibe work，再到 vibe management 的路径。

工作不会消失。工作的界面会消失。

剩下的是意图、信息流和智能系统。
